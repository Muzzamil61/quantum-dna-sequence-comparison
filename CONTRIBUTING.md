# Contributing to Quantum DNA Sequence Comparison

We welcome contributions to the Quantum DNA Sequence Comparison project! Your input helps us improve and expand the project.

## How to Contribute

1. **Fork the Repository**: Click the "Fork" button on the top right corner of the repository page to create a copy of this repository on your GitHub account.

2. **Clone Your Fork**: Clone the forked repository to your local machine.
    ```bash
    git clone https://github.com/yourusername/quantum-dna-sequence-comparison.git
    cd quantum-dna-sequence-comparison
    ```

3. **Create a New Branch**: Create a new branch to work on your feature or bug fix.
    ```bash
    git checkout -b feature/your-feature-name
    ```

4. **Make Changes**: Add your changes or new features to the codebase. Ensure that your changes are properly documented and adhere to the project's coding style.

5. **Test Your Changes**: Run the scripts and Jupyter notebooks to ensure your changes do not break existing functionality.

6. **Commit Your Changes**: Commit your changes with a meaningful commit message.
    ```bash
    git commit -m "Add a descriptive commit message"
    ```

7. **Push to Your Fork**: Push the changes to your forked repository.
    ```bash
    git push origin feature/your-feature-name
    ```

8. **Create a Pull Request**: Go to the original repository on GitHub and click on the "Pull Requests" tab. Click "New Pull Request" and select your branch from your fork. Write a detailed description of your changes and submit the pull request.

## Guidelines for Contributions

- **Code Style**: Follow Python's PEP 8 style guide for clean and readable code.
- **Documentation**: Ensure that your code is well-documented with comments and that any new features are described in the `USAGE.md` or other relevant documentation files.
- **Testing**: Ensure that your changes are tested thoroughly and do not break existing functionality.
- **Issues**: Reference any issues your pull request addresses in the description (e.g., "Fixes #123").

## Reporting Bugs and Feature Requests

If you find a bug or have a feature request, please open an issue in the GitHub repository. Provide a detailed description of the problem or feature and include any relevant information to help us understand the context.

## Community and Support

For discussions, questions, and support, feel free to join our community discussions in the "Discussions" tab of the repository.

We appreciate your contributions and efforts to help make this project better!
